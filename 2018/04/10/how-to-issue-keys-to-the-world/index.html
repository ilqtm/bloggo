whoops

Last night while (needlessly) trying to use git to post hexo updates to track commits, I managed to post my aws keys.
I realized immediately and disabled them in the AWS Console, issued new ones, reset my _config.yml, definitely was sure I set things up again correctly…

And did it again.

It’s an easy mistake to make, I’m glad I made it in a low stakes environment, but it’s easy to avoid too. In the aftermath I set up newer keys (deleted the old sets), applied them to the necessary files, and realized that hexo now shows commits to my github correclty.
Right now I have it deploying changes to github (w/ hexo-deployer-s3), but all the actual hosting and dns is through AWS. s3 to host and Route 53 to serve.

This kind of thing can be avoided by just leaving them out w/ the first commit to set up the repo and then adding wherever they end up into your .gitignore. I’m just going to avoid it for now until I know more.

My goal yesterday was to get a CloudFront distribution going to see what the set up is like. I got the distribution set up, that’s easy (it’s a wizard, Harry!), I just keep getting 403’d. I’ll spend some time on it tonight to see if it’ll go, tomorrow if it doesn’t.

Oh, an unforseen consequence of unintentionally posting my keys twice? Two:

  • GitGuardian is a neat whitehat security bot that scans files in GitHub for keys, then emails you about it listing the repo they’re in (and that you probably should remove them)
  • Waking up and logging in to AWS multiples times to check my account wasn’t being klepto’d for crypto mining or botnets

ethical scripting

I don’t like paywalls. Duh, nobody outside of one wanting what’s inside does. I’m not stepping out on a limb here.

I’m actively considering the ethics of breaking them in order to learn about security and web developement in practice. Obviously, this also gets me the content I’m interested in w/o paying/membership, but oddly is something I discovered my mistake.
While writing loops for some of my favorite recipe sites to avoid the often long winded blog posts (imo), I applied the script out of habit to a site that loaded an “n articles allowed for non-members, login/pay to see content”.

A new tab opened w/ the desired plaintext recipe and method, regardless of the paywall popup.

Hmm. I’m not the first one to realize or do this, I’m sure. I know this is borderline script kiddie territory, but it got me curious, so I’m trying it with other sites I enjoy and it works w/ a little tinkering.
Should the authors/devs be told? Would they care? Do they already know? Is it inevitable for vulerabilities like this to be exploited?

It just doesn’t take much, I guess.
