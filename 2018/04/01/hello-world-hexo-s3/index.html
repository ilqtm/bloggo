Deploy to remote sites

More info: Deployment

Here is where I had a little snag. Using hexo-deployer-s3 is great since the idea was to host on s3. However, there aren’t instructions listed for it on the official documentaion. All one needs to do is edit the _’config.yml’ wherever you did

and add in

deploy:
type: s3
bucket:
aws_key:
aws_secret:
region:

Bucket: is literally the name of the S3 bucket you want Hexo deploying to. I got it wrong a few times w/ these formats:

  • arn:aws:s3:::ilqtm.net
  • s3.us-west-2.amazonaws.com/ilqtm.net
  • s3-us-west-2.amazonaws.com/ilqtm.net
  • ilqtm.net.s3-us-west-2.amazonaws.com

When it goes wrong Hexo spits out an error saying the bucket doesn’t exist or that the Endpoint needs to be used; hence the use of endpoints after the ARN didn’t work. Here’s what does work:

ilqtm.net

The barebones, inside the S3 console, AWS S3 ls, plain name of the bucket.

The aws_key and aws_secret are listed as optional (along w/ a few other perameters) if you’ve already configred the aws-cli, but I wasn’t able to get Hexo to deploy w/out them in the _config.yml.

So long.
