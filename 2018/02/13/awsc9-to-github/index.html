Uninterpretable symbol combinations, apparent function errors, and a permission slip

The attempt and then (partial) success to deploy to aws s3 and github simultaneously from aws cloud 9 is coming together. At this point I can get either to work, but not yet both w/ one deploy.
I’ll do the old comment switcheroo for now and just deploy separately.

My first issue an extra doublequote that was understandably missed w/o my glasses on.
The error showed as:

Template render error: (unknown path)

Second was a whitespace (WS) issue that caused a duplicate key error. YAML needs specific whitespace indentation and dashes for lists that contain multiple instances of the same key.

incorect:

deploy:
type: s3
type: git

incorrect:

deploy:

  • type: s3
  • type: git

correct:

deploy:
WS- type: s3
WS- type: git

Third was a function apparently called ‘now’ that was deriving from a JSON expression I put into the deploy section of _config.yml. It’s the default value for the key, so there wasn’t much point including it.

Last, so far, was a permissions issue. I didn’t set up the public key! There is a public key already on the ec2 that hosts the cloud 9 IDE, but it didn’t work, so I made a new one, applied it to the github account, corrected my repo URL, and boom bam.
